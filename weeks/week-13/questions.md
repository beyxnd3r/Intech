# Вопросы для самопроверки

1. Зачем нужен Helm? Какую проблему он решает?
   Это пакетный менеджер для Kubernates. Он решает проблему дублирования yaml
2. Как работает приоритет значений (Values)? Кто победит: `values.yaml` или флаг `--set`?
   Изначально values потом dev/prod. Если есть --set то в приоритете будет он.
3. Что такое функция `include` и `_helpers.tpl`? Зачем они нужны?
   include вызывает шаблонные функции из helpers. Они нужны для переиспользвоания кода и для избавления от дублирования кода
4. Почему в шаблонах Helm так важно следить за отступами и зачем нужна функция `indent` или `nindent`?
   Yaml чувствителен к отступам. Функции нужны для отсутпа и отсутпа с переносом строки   
5. Как откатить неудачный релиз с помощью Helm?
   helm history my-release и потом helm rollback release revision
6. Можно ли использовать Helm только для генерации YAML, без установки в кластер?
   Да можно, так даже удобнее для откладки
