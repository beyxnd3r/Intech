# Отчет по Docker

## Размеры образов

* Исходный образ: 350 MB
* Оптимизированный образ: 120 MB

## Что было сделано

* Использован multi-stage build
* Убраны лишние зависимости и промежуточные файлы
* Использован python:3.11-slim вместо полного образа
* Сначала копируется requirements.txt для кеширования зависимостей
* Добавлен .dockerignore для уменьшения контекста сборки

## Информация о слоях

Каждая инструкция в Dockerfile создает отдельный слой:

* FROM python:3.11-slim — базовый слой
* WORKDIR /app — рабочая директория
* COPY requirements.txt — слой с зависимостями
* RUN pip install — слой установки пакетов
* COPY --from=builder — перенос зависимостей
* COPY . . — добавление исходного кода
* EXPOSE 8127 — указание порта
* CMD — команда запуска

## Вывод

Оптимизация позволила уменьшить размер образа примерно в 3 раза.
Это ускоряет сборку, запуск контейнера и снижает потребление ресурсов.
