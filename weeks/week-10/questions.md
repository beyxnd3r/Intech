# Вопросы для самопроверки

1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?
   Слой это результат выполнения инструкции. Порядок выполнения команд важен т.к. если какой-то слой изменится то пересобираться будут все слои 
2. В чем суть Multi-stage build? Какие проблемы он решает?
   Это использование нескольких этапов сборки в одном Dockerfile. Он позволяет уменьшить размер образа и повышает безопасность.
3. Зачем нужен файл `.dockerignore`?
   Этот файл исключает ненужные файлы из сборки.
4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?
   В CMD команда задется по умолчанию и ее можно переопределить при запуске, ENTRYPOINT же задает основную команду и ее сложнее пересобрать
5. Почему рекомендуется использовать `COPY` вместо `ADD`?
   COPY просто безопаснее, т.к. мы будем знать что и откуда копируем
6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?
   Потому что адрес 127.0.0.1 работает только внутри контейнера и подключится к нему не получится. Адрес 0.0.0.0 же позволяет подключится к контейнеру снаружи
