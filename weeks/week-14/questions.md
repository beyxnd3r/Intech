# Вопросы для самопроверки

1. В чем разница между Continuous Integration, Continuous Delivery и Continuous Deployment?
   Integration - это практика, при которой разработчики часто сливают код в репозиторий, а система автоматически запускает тесты и проверки,
   Delievery - код после CI автоматически готовится к релизу (собирается, тестируется), но выкладка в прод происходит вручную
   Deployment - полностью автоматический процесс: код после всех проверок сразу выкатывается в прод без участия человека
2. Почему тесты должны запускаться *до* сборки Docker образа (или хотя бы до его пуша)?
   Чтобы не тратить время на нерабочие сборки и чтобы не создавать сломаные образы
3. Зачем нужно кэширование зависимостей в CI пайплайне?
   Оно ускоряет сборку, уменьшает нагрузку на сеть и делает pipeline стабильнее
4. Что такое "Secret" в контексте CI/CD? Как безопасно передать пароль от Docker Hub в пайплайн?
   Secret - это скрытые данные не хранящиеся в коде. Безопасно можно передать через настройки на сайте GitHub
5. Что такое "Matrix Build" (матричная сборка)?
   Matrix Build - это запуск пайплайна в нескольких вариантах одновременно для проверки разных условий
6. Почему нельзя хранить конфиги продуктовой среды в репозитории кода (или можно, но осторожно)?
   Нельзя так как может быть риск утечки или взлома
