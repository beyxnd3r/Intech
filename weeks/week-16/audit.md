## 4. Найденные уязвимости

### 4.1 Отсутствие аутентификации
Риск: HIGH  
Описание: API доступен без проверки пользователя  
Влияние: любой пользователь может выполнять запросы  
Решение: внедрить JWT авторизацию  

---

### 4.2 Отсутствие авторизации
Риск: HIGH  
Описание: нет проверки владельца ресурса  
Влияние: доступ к чужим данным  
Решение: проверять user_id  

---

### 4.3 Отсутствие валидации данных
Риск: MEDIUM  
Описание: принимаются любые данные  
Влияние: возможны ошибки и атаки  
Решение: использовать схемы (Pydantic/Protobuf)

---

### 4.4 Нет rate limiting
Риск: MEDIUM  
Описание: сервер не ограничивает количество запросов  
Влияние: возможен DoS  
Решение: добавить ограничения (например, 100 req/min)

---

### 4.5 Нет HTTPS
Риск: MEDIUM  
Описание: данные передаются открыто  
Влияние: возможен перехват  
Решение: использовать HTTPS

---

### 4.6 Возможность XSS
Риск: LOW  
Описание: данные не экранируются  
Влияние: внедрение скриптов  
Решение: экранирование входных данных  

---

### 4.7 Логирование чувствительных данных
Риск: MEDIUM  
Описание: возможна утечка через логи  
Влияние: компрометация данных  
Решение: фильтрация логов  

---

## 5. Попытки атак

### SQL Injection
Запрос:
' OR 1=1 --
Результат: не применимо (нет SQL базы)

---

### Неавторизованный доступ
Попытка:
- запрос без токена  

Результат:
- доступ получен 

---

### DoS атака
Попытка:
- большое количество запросов  

Результат:
- деградация производительности 

---

## 6. Анализ рисков

Основные угрозы:
- несанкционированный доступ (HIGH)
- перегрузка сервера (MEDIUM)
- утечка данных (MEDIUM)

---

## 7. Рекомендации

1. Внедрить JWT аутентификацию
2. Реализовать авторизацию
3. Добавить rate limiting
4. Использовать HTTPS
5. Валидация входных данных
6. Защитить логи
7. Обновлять зависимости

---

## 8. Вывод

Сервис имеет базовые уязвимости, которые свойственны начальным проектам.
Основные проблемы — отсутствие аутентификации и защиты от нагрузки.

После внедрения рекомендаций уровень безопасности значительно повысится.
