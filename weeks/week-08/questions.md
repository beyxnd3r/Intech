# Вопросы для самопроверки

1. Что такое Server Streaming? Приведите пример использования.
   Это режим gRPC, где клиент отправляет 1 запрос, а сервер возвращает поток ответов. В пример можно привести мониторинг чего либо.
2. Чем Server Streaming отличается от просто получения большого списка (List) в одном ответе?
   При Server Streaming данные отправляются не разом а частями.
3. Какие метрики производительности существуют (Latency, Throughput, CPU Usage)?
   Задержка, пропускная способность, Использование процессора, использование памяти и др.
4. Почему важно переиспользовать канал (Channel) в gRPC клиенте при замерах производительности?
   Для сохранения целотсности данных и стабильной производительности
5. В каких сценариях REST может оказаться быстрее или удобнее gRPC?
   Там где нет его поддержки или например в создании веб приложений
6. Что такое "Head-of-Line Blocking" и как HTTP/2 (и gRPC) решает эту проблему?
   Это блокировка запросов одним медленным запросом. gRPC и HTTP/2 решают эту проблему путем использования мультиплексирования.
