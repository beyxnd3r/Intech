# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?
Forward Proxy:
работает от имени клиента
скрывает клиента от интернета

Пример: корпоративный прокси

Reverse Proxy:
работает от имени сервера
скрывает реальные сервисы

Пример: Nginx перед микросервисами.
1. Что такое Upstream в терминологии Nginx?
Upstream — это группа бэкенд-серверов, куда прокси пересылает запросы.
2. Зачем скрывать реальные IP-адреса сервисов от клиента?
Для безопасности, для защиты от взломов и атак
3. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?
Аутентификация/Авторизация, Ограничение частоты запросов
4. Что произойдет, если Gateway упадет? Как можно от этого защититься?
Использовать несколько образцов gateway, использовать DNS или же сервисы такие как Cloudflare для предотвращения и балансировки
