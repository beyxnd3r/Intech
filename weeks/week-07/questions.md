# Вопросы для самопроверки

1. Зачем нужен файл `.proto`? Почему мы не пишем классы сообщений сразу на Python?
   Proto это описание API который можно использовать для любого языка
2. Что такое Unary RPC? Чем он концептуально похож на REST запрос?
   Это тип взаимодействия gRPC, когда отправляется один запрос и сервер получает один ответ. Также как REST имеет синхронное взаимодействие.
3. Почему gRPC обычно эффективнее REST (JSON) по процессору и трафику?
   Так как используется Protobuf вместо json то он имеет меньший размер, также он может отправлять несколько запросов по одному соединению. Это позволяет снизить нагрузку на трафик и на процессор
4. Что такое "обратная совместимость" в Protobuf и зачем нужны номера полей (тэги, например `= 1`)?
   Это способность новой ситсемы proto работать со старой. Номера полей нужны для идентификации.
5. Что такое Stub (стаб) на клиенте и Servicer на сервере?
   Stub это прокси который клиент вызывает на сервер, Service - это реализация методов сервиса.
6. В каких случаях вы бы выбрали REST, а в каких — gRPC?
   REST - для бэкенда, вебсайтов. gRPC - для микросервисов.
