# Ответы на вопросы

## 1. Какие критерии вы использовали при разделении монолита на микросервисы (или при проектировании сервисов)?

Я разделил систему по зонам ответственности:  

- **events-service** - работа с API и событиями  
- **grpc-service** - внутренняя логика  
- **gateway** - вход и маршрутизация  

Критерии: разделение логики, независимость сервисов, возможность масштабирования  

---

## 2. Почему вы выбрали именно этот стек технологий (язык, БД, протоколы)?

- Python + FastAPI - быстро писать REST API  
- gRPC - быстрый обмен между сервисами  
- Docker - одинаковая среда запуска  
- Nginx - простой gateway  

---

## 3. Как ваши сервисы справляются с ошибками (retry, circuit breaker, graceful degradation)?

Сейчас есть базовая обработка через HTTP ответы, fallback  

---

## 4. Как организован деплой и обновление вашей системы без простоя (Zero Downtime)?

Через Helm в workflow  

---

## 5. Что было самым сложным в интеграции всех компонентов вместе?

Настраивание Docker, Kubernetes и связывание, пусть проcтых, но все же небольших разных сервисов вместе  

---

## 6. Что бы вы улучшили, если бы у вас был еще месяц?

Добавил бы нормальную базу данных, добавил бы авторизацию, улучшил обработку ошибок
